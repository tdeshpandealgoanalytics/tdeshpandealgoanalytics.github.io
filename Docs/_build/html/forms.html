forms module¶

This forms() moduleplays one of the crucial roles in Aksha. This module contains all the functions related to Cameras on our UI. This module has the functionalities to add a new camera, to update an existing camera’s configuration or to view configurations of an existing camera.

class forms.AddCamera(parent, app_font)¶

Bases: tkinter.Toplevel

submit(event)¶
class forms.UpdateCamera(parent, app_font)¶

Bases: tkinter.Toplevel

The UpdateCamera() class uses tkinter library of python in order to provide the user with an interface which helps them to Update any Camera’s information into our database. It provides a list of various buttons such as select camera, update camera, new name etc. onto the screen.

show_restart_messagebox()¶

This function is used in the above class (UpdateCamera()) in order to display a restart message to the user when a Camera is updated

submit(event)¶

This function provides a Submit button to the user which reads and finally updates all the information at their appropriate places. For this it takes help from the edit_camera() function from camera_database.py

update_fields(event)¶

This function updates the existing fields inside the UpdateCamera() function

class forms.ViewCameras(parent, app_font, width, height)¶

Bases: tkinter.Toplevel

The ViewCameras() uses the tkinter library which displays critical information regarding all the cameras in the database. The information like camera name and their respective URL is displayed to the user.

onFrameConfigure(canvas)¶

This function is used in the ViewCameras() class in order to reset the scroll region to encompass the inner frame
